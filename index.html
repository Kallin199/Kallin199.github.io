在linux环境下配置MoJoco与dm_control环境

DeepMind Control Suite 是 DeepMind 最新开源的，一套有标准化结构的持续控制任务，旨在成为强化学习 Agent 的性能基准。Control Suite 由 Python 编写，并由 MuJoCo 物理引擎驱动。

1. 安装MuJoco

  • 安装包下载：DeepMind开源并维护了MoJoco200后的版本安装地址，而MuJoco200以及之前的版本可以在地址下载

  • 激活文件下载：MuJoco开源以后，激活码可以直接在官网下载下载地址

  • 文件布置：在根目录创建文件夹.mujoco，里面存放解压后的安装包mujoco200和激活码mjkey.txt。同时将激活码复制到mujoco200下的bin目录下

  • 添加环境变量vim ~/.bashrc，加入如下地址后运行source ~/.bashrc生效

  • 测试：若出现仿真画面或者显示MuJoco版本MuJoCo Pro version 2.00则安装成功

2. 安装MoJoco_py

  • pip安装：pip3 install -U 'mujoco-py<2.2,>=2.1'
  • 测试：进入python环境，import mujoco_py，若无报错则安装成功

3. 安装dm_control 库

在确保MuJoco安装成功后，进行dm_control项目地址的安装，有两种安装方式可供选择：

  1. pip install dm_control （默认安装的版本适用于MuJoco211）
  2. pip install git+git://github.com/deepmind/dm_control.git

安装成功后，可以进入python环境import dm_control进行测试。

踩坑记录

报错一：dm_control直接安装

ImportError: cannot import name ‘constants’ from partially initialized module ‘dm_control.mujoco.wrapper.mjbindings’ (most likely due to a circular import) (/home/lzx/dm_control/dm_control/mujoco/wrapper/mjbindings/init.py)

原因：如果通过下载源文件+pip install -e .的方式安装，则dm_control被安装在文件下载的路径，而非 python package folder。

解决办法：通过指定版本安装对应的dm_control： pip install dm-control==0.0.364896371

报错二：指定环境路径

使用hexo进行博客搭建

发表于 2022-02-02

环境配置

使用hexo在本地可以方便、快速地搭建个人博客，同时依赖github可实现将个人博客推送至远端，本篇介绍在Macbook上利用hexo和node.js搭建属于自己的博客

  1. 安装node.js

    在官网下载并按步骤安装(安装完成后会一并配置npm包管理器)

  2. 安装hexo

命令行输入：npm install -g hexo-cli

环境配置验证

命令行输入node -v, hexo -v, npm -v检测是否安装成功

本地博客搭建

  1. 在本地创建一个文件夹用于存放博客的相关文件(存放位置自定义)

​ 命令行输入：makir blog

  1. 进入blog文件，使用hexo初始化博客

    命令行输入：hexo init

​ 初始化完成后会看到提示: start blogging with hexo

  1. 使用本地预览命令启动博客

    命令行输入：hexo s

  2. 新建一篇博文

​ hexo -n "blog name"，之后可以使用任意markdown编辑器书写博文

  1. 生成博文

​ hexo g

随后便可在本地博客查阅新建的博文。

部署博客

推荐使用github部署博客

  1. 在github新建仓库，注意仓库名称必须为github用户名，即<username>.github.io。
  2. 在本地blog文件下安装插件npm install --save hexo-deployer-git
  3. 打开blog/_config.yml文件，增加Deployment部分
  1. 部署到远端

​ hexo d

  1. 访问博客，使用https://username.github.io

这样一个属于自己的博客就建成啦！

可能遇到的问题

密钥配置

​ 如果之前没有在github上提交过项目，则需要配置密钥

  1. 设定git账户和邮箱，其中 username 是你的用户名，email@gmail.com是你github的登录邮箱
  1. 然后通过终端命令生成SSH Key
  1. 复制SSH Key
  1. 打开并登陆github，点击用户头像>settings>SSH and GPG keys>New SSH key 粘贴刚刚复制的密钥，就可以将本地项目推送至远端啦。

主题配置

​ 在本地我们可以到hexo的主题官网hexo-themes下载喜欢的主题，并添加至本地blog/theme文件夹中，并在_config.yml文件中修改配置，使得主题生效。
