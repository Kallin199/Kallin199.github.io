tmux常用命令

tmux采用C/S模型构建，输入tmux命令就相当于开启了一个服务器，此时默认将新建一个会话，然后会话中默认新建一个窗口，窗口中默认新建一个面板。会话、窗口、面板之间的联系如下：

一个tmux session（会话）可以包含多个window（窗口），窗口默认充满会话界面，因此这些窗口中可以运行相关性不大的任务。

一个window又可以包含多个pane（面板），窗口下的面板，都处于同一界面下，这些面板适合运行相关性高的任务，以便同时观察到它们的运行情况。

运行tmux后，即使与服务器中断连接，程序也可以继续运行，其现场保护功能大大方便了学习和工作

invariant representations

Zhang A, McAllister R, Calandra R, et al. Learning invariant representations for reinforcement learning without reconstruction[J]. arXiv preprint arXiv:2006.10742, 2020.

痛点问题

  1. 状态编码是否与策略学习相关

    • 自动编码器：基于重构样本，即希望从学习到的表示出发能够尽可能重构原本的状态。

    • 对比学习：只是希望表示中包含能够与其他状态分辨开的关键信息。尽管 取得了比较好的效果，大多数都利用了状态是图片这样一个先验知识

    • 预测学习：捕捉状态中的可预测特征，基于真实图像表现不佳

    • 互模拟（Bisimulation）：使用互模拟矩阵学习RL结构特征，难以泛化到高维表示

算法思路

​ 训练完成后使用双模拟度量（左 t-SNE）和 VAE（右 t-SNE）学习的潜在空间的 t-SNE，用预测状态值（较高值黄色，较低值紫色）进行颜色编码。

主要贡献

为了判断两个状态的相似程度，基于两者的后续单步reward最大差距以及后续跳转状态的差异。

可视化的结果表明该方案能够自动学到和task相关的表征信息，在经典的benchmark上效果也超越了之前的主流方案。

Distance Learning

Hartikainen K, Geng X, Haarnoja T, et al. Dynamical distance learning for semi-supervised and unsupervised skill discovery[J]. arXiv preprint arXiv:1907.08225, 2019.

解决痛点

  1. 奖励函数设计困难，虽然传统Q-learning能够在低维空间学习距离，但是难以学习高维环境
  2. 适用于复杂问题的高维状态（pixel）

主要工作

  1. 采取最短路径到达目标的策略，产生有效的定向探索
  2. 使用自动学习当前状态到理想状态的动态距离，这些动态距离可用于为实现新目标提供形状良好的奖励功能，从而可以高效学习复杂的任务。

算法原理

  1. 监督训练动态距离，$s_i,s_j\in \tau_k$

$$
\mathcal L_d(\phi) = \frac{1}{2} \mathbb E_{\tau, i,j}[(d^{\pi}_{\phi}(s_i,s_j)-(j-i))^2]
$$

2.优化目标为最小化累计目标距离, $g \in goal$，采用累计距离是为了平衡状态转移的不确定性，若优化贪婪距离，可能会因为环境变化导致训练难以收敛。（上次由于随机因素到达指定的后续状态。）
$$
\mathcal L_{\pi}(\phi)=\mathbb E_{\tau} [\sum^\inf {t=0} \gamma^t (d^{\pi}{\phi}(s_i,g)]
$$

RL-论文泛读：Dynamics-aware unsupervised discovery of skills.md

发表于 2022-02-17 分类于 reinforcement learning

DADS

Sharma A, Gu S, Levine S, et al. Dynamics-aware unsupervised discovery of skills[J]. arXiv preprint arXiv:1907.01657, 2019.

主要贡献

  1. 结合model-free进行预训练，使用model-based得到策略。
    • 连续技能空间
    • 可以处理稀疏奖励

算法原理

优化互信息,获得无监督技能z，可被认为是一系列动作集合：
$$
I(s’;z|s)=\mathcal H(z|s)-\mathcal H(z|s’,s) = \mathcal H(s’|s)-\mathcal H(s’|s,z)
$$

  • $\mathcal H(s’|s)$：使得已知当前状态情况下，下一时刻状态尽可能多样

  • $\mathcal H(s’|s,z)$：引入技能z以后，下一时刻状态尽可能易预测

最终目标是优化基于技能的策略$\pi(a|s,z)$,其中采样的$z \sim p(z)$（由用户事先定义）可以最大化的提供状态转移信息。

智能体与环境交互产生转移数据，通过估计$q(s’|s,z)$的预测正确性来获得奖励

状态转移估计器通过最大化真实转移概率进行更新

技能z视为对一系列动作集合在隐空间$Z$的编码，使得智能体对前序行为进行总结，然后使用技能z和当前状态预测后续状态转移。

Latent Space Planner: Planner维护隐策略空间的奖励分布，使用状态转移函数对各个skill下的状态转移轨迹进行模拟，计算奖励

  1. 训练时，更新奖励参数$\mathcal N_i(\mu_i,\sum)$.

  2. 测试时，使用具有最高优先级的子策略进行行动

策略使用正态分布（均值为奖励，标准方差）进行编码。每个技能的视距为$H_Z$, ，共有$H_P$个隐变量。

专家评注

​ 学习用于模型预测控制的低级“技能”，通过最大化下一个状态s和当前状态s的当前技能z之间的相互信息来学习的，相当于最大限度地扩大环境中产生的过渡的多样性，同时使技能z为下一个状态提供信息。

​ 技能z对应于一组动作序列，这些动作序列由分布$\pi(a|s,z)$表示，使用近似动力学$q_{\phi}(s’|s,z)$变分界模拟转移函数，表示使用技能z在当前状态的环境转移，优化该变分下界以产生最佳$q_{\phi}(s’|s,z), \pi(a|s,z)$。在奖励阶段，使用学习的过渡动力学和技能函数模拟多个轨迹，根据奖励函数计算每个轨道的奖励，执行最佳轨道的第一个动作并重复。规划是在潜在的技能空间中完成的，这使得更轻松的长视距规划成为更容易。

（1）学习的技能表现出低方差行为（这意味着这些技能在用于模型预测控制时具有可预测的行为）

（2）模型预测控制与其他相关基线相比表现良好。

VISR

Hansen S, Dabney W, Barreto A, et al. Fast task inference with variational intrinsic successor features[J]. a conference paper at ICLR 2020

问题引入

  1. 在无监督环境中训练成本较低，此阶段智能体自由地和环境交互，收集环境的信息，以尽可能加快强化学习阶段。

  2. 泛化能力高，智能体可以根据奖励的设置执行定义中的一项或者多项任务

    由于无监督学习预训练模型独立于奖励，最大化潜在变量策略与状态访问提取的行为信息

痛点问题

  1. 传统无监督模型泛化性能差：

    以前无监督预训练RL模型（DIAYN、VIC）在适应下游任务时，泛化性差、推理过程缓慢的。下游任务可能在无监督学习的标准行为之间，所以需要潜在行为空间进行插值。构建训练期间未遇到的潜在行为的策略时此方法的主要问题。

主要贡献

  1. 利用后继特征，假设奖励函数在某些特征中是线性的，将奖励作为特征的加权求和。后继特征支持仅奖励函数不同的任务之间进行快速迁移学习。

算法原理

  • 在无监督训练阶段，学习到各个状态动作对的后继特征，将后继特征总结为一个网络
  • 在监督阶段，通过拟合奖励学习$w$，（通过实际奖励定义哪个状态特征好，哪个状态差）
  • 在监督学习阶段，由于不知道$s’$，所以需要通过子策略来进行预测，拟合奖励生成n个子策略，在后继特征网络中输入$s,a,e(\pi_i)$，输出当前子策略$\pi_i$在$(s,a)$下的后继特征，与权重向量相乘以后，组合出Q和最终策略，由子策略组合而成

如何提取状态特征-子策略学习？
$$
F(\theta) = I(z;f(\tau_{\pi_{\theta}}))=H(z)-H(z|f(\tau_{\pi_{\theta}}))
$$
最大化子策略与代理行为之间的互信息，$f$输出采样当亲轨迹中的某个状态。

APT

Liu, H. and Abbeel, P. Behavior from the void: Unsuper- vised active pre-training. In International Conference on Machine Learning, 2021.

主要贡献

  1. 使用对比学习获得低维度状态特征
  2. 通过最大化在抽象表示空间中计算的非参数熵（k近邻距离）来探索环境，这避免了具有挑战性的密度建模。
  3. 一方面最大化期望奖励，一方面降低表示误差。使用特征空间中的k近邻距离作为内部奖励：

总结

  1. 使用像素状态作为输入
  2. 不需要维护密度估计器，使用k近邻
  3. 使用对比学习对状态进行编码
  4. 表现得好的前提是，内部奖励和外部奖励处于同一数量级。后续改进
  5. fine-tuning部分讲的不是很清楚。为什么要匹配奖励数量级，外部奖励如何使用？
  6. 在fine-tuning过程中容易产生灾难性遗忘。解决方式：固定参数与随机初始化相结合

APS

Liu H, Abbeel P. Aps: Active pretraining with successor features[C]//International Conference on Machine Learning. PMLR, 2021: 6736-6747.

问题引入

第一类基于最大化任务变量 (p(z)) 及其在状态访问 (p(s)) 方面的行为之间的互信息，以鼓励学习可区分的任务条件行为，这已被证明是有效的 在基于状态的强化学习和视觉强化学习中。 Hansen 等人提出的 VISR是该类别中最先进的技术，其无需进行密度估计。VISR 的优势在于其后续功能可以快速适应新任务。 尽管它很有效，但 VISR 面临的根本问题是缺乏探索。

第二类是最大化状态熵。APT就是此类算法的典型代表，使用最近邻距离最为奖励。然而，APT 的问题在于它纯粹是探索性的和任务不可知的，并且缺乏任务变量的概念，与任务调节策略相比，它更难适应新任务。

RL-无监督测试基准.md

发表于 2022-02-17 分类于 reinforcement learning

URLB

Laskin M, Yarats D, Liu H, et al. URLB: Unsupervised Reinforcement Learning Benchmark[J]. Conference on Neural Information Processing Systems, 2021.

问题引入

​ 强化学习能解决众多复杂但是具体的控制问题，但是训练具有泛化能力的智能体，使其能迅速适应新任务仍然缺乏研究。在过去的五年中，深度强化学习 (RL) 一直是自主控制领域多项突破的源泉。 RL 算法已被用于训练代理玩 Atari 直接从像素 [44, 45] 学习视频游戏，学习机器人运动 [52-54] 和操纵 [2] 来自原始感官输入的策略，掌握围棋 [58, 59] 并玩大型多人游戏 视频游戏 [6, 65]。 虽然这些结果是自主决策的重大进步，但更深入的观察揭示了一个基本的限制。 上述算法产生的代理只能 解决他们被训练解决的单一任务。 因此，当前的 RL 方法会产生脆弱的泛化能力差的策略[16]，限制了它们对许多问题的适用性感兴趣 [23]。 因此，重要的是要超越当今强大但狭窄的 RL 系统 面向能够快速适应新下游任务的通用系统

模型介绍

​ 监督学习执行每一步动作都需要环境奖励，而无监督学习模型具有两个部分：无奖励预训练和基于外部奖励微调，预训练模型的优劣由下游任务的快速适应性衡量

基准介绍

  1. 状态表示：states 或是像素信息，分离表证学习变量

  2. 在预训练阶段，在智能体训练的100k，500k，1M，2M处设置断点进行测试

  3. 在下游测试阶段，以100k的频率进行测试评估

  4. 测试环境： DeepMind Control Suite(DMC),广泛采用，两种obs输入，具有不同难度的连续动作环境

评价基准：1.能快速适应下游任务的策略被定义为好的，2.测量数据多样性

控制变量：由于强化学习过程包含：表征学习+行为决策。所以为了去除表征学习对策略表现的影响，设计基于状态和基于图像提取的特征来进行行为决策。实现解耦

说明：基本RL模型的统一，由于SAC容易造成策略熵崩溃，使用DDPG比SAC训练出的策略更具鲁棒性

未来展望

  1. 已实现的无监督 RL 算法均无法解决基准问题。

    尽管访问最多 到 2M 预训练步骤，经过 100k 步微调后，没有方法与渐近性能相匹配 在大多数任务上。 性能最佳的基准算法实现了 40 - 70% 的归一化回报 而当代理达到接近 100% 的归一化回报时，基准被认为已解决。 这表明我们距离深度强化学习中的有效泛化还很远

  2. 无监督 RL 并非普遍优于随机初始化。

    我们还观察到微调无监督的 RL 基线并不总是优于从随机的微调初始化。 特别是在从状态中学习时，随机初始化与大多数基线。 然而，当从随机初始化的像素微调中学习时 表明表征学习是无监督预训练的重要组成部分。

  3. 从状态探索和从像素探索之间存在很大的性能差距。

    支持表征学习作为探索的一个重要方面的另一个观察结果是 与学习相比，从像素学习时，探索算法会显着降低 从状态。 如图 3 所示，相比之下，大多数算法在从像素中学习时损失了 20 - 50% 说明，尤其是在更困难的环境中（四足动物，Jaco Arm）。 这些结果表明 在预训练期间更好的表征学习是一个重要的研究方向

  4. 总的来说，基于能力的方法不如基于知识和数据的方法 方法。

    虽然基于知识和基于数据的方法在跨领域都具有竞争力 URLB，我们发现基于能力的方法落后了。 具体来说，没有 基于能力的方法，可在任何 URLB 上实现最先进的平均性能 任务，它指出基于能力的无监督 RL 作为一个有影响力的研究方向 很大的改进空间

  5. 没有一个基准同时在基于state和基于图像的特征上表现最优

  6. 当前无监督模型的最大问题：fine-tune的效率随着预训练模型而改进，目前这两部分是相互分离的。期望看到fintune’后的表现随着预训练步数的增加而增加。

  7. 需要新的微调策略来快速适应任务。目前是将预训练权重初始化AC，并且通过外在奖励进行微调，可能还有其他更好的微调策略。特别是针对SMM这类算法

预训练模型中存储有当前模型具有的能力z，若下游任务能被快速判断为当前任务的子集，则更好的微调模型就可以被得到。

在linux环境下配置MoJoco与dm_control环境

DeepMind Control Suite 是 DeepMind 最新开源的，一套有标准化结构的持续控制任务，旨在成为强化学习 Agent 的性能基准。Control Suite 由 Python 编写，并由 MuJoCo 物理引擎驱动。

1. 安装MuJoco

  • 安装包下载：DeepMind开源并维护了MoJoco200后的版本安装地址，而MuJoco200以及之前的版本可以在地址下载

  • 激活文件下载：MuJoco开源以后，激活码可以直接在官网下载下载地址

  • 文件布置：在根目录创建文件夹.mujoco，里面存放解压后的安装包mujoco200和激活码mjkey.txt。同时将激活码复制到mujoco200下的bin目录下

  • 添加环境变量vim ~/.bashrc，加入如下地址后运行source ~/.bashrc生效

  • 测试：若出现仿真画面或者显示MuJoco版本MuJoCo Pro version 2.00则安装成功

2. 安装MoJoco_py

  • pip安装：pip3 install -U 'mujoco-py<2.2,>=2.1'
  • 测试：进入python环境，import mujoco_py，若无报错则安装成功

3. 安装dm_control 库

在确保MuJoco安装成功后，进行dm_control项目地址的安装，有两种安装方式可供选择：

  1. pip install dm_control （默认安装的版本适用于MuJoco211）
  2. pip install git+git://github.com/deepmind/dm_control.git

安装成功后，可以进入python环境import dm_control进行测试。

报错一：dm_control直接安装

ImportError: cannot import name ‘constants’ from partially initialized module ‘dm_control.mujoco.wrapper.mjbindings’ (most likely due to a circular import) (/home/lzx/dm_control/dm_control/mujoco/wrapper/mjbindings/init.py)

原因：如果通过下载源文件+pip install -e .的方式安装，则dm_control被安装在文件下载的路径，而非 python package folder。

解决办法：通过指定版本安装对应的dm_control： pip install dm-control==0.0.364896371

使用hexo进行博客搭建

发表于 2022-02-02 更新于 2022-02-18 分类于 Hexo

环境配置

使用hexo在本地可以方便、快速地搭建个人博客，同时依赖github可实现将个人博客推送至远端，本篇介绍在Macbook上利用hexo和node.js搭建属于自己的博客

  1. 安装node.js

    在官网下载并按步骤安装(安装完成后会一并配置npm包管理器)

  2. 安装hexo

命令行输入：npm install -g hexo-cli

环境配置验证

命令行输入node -v, hexo -v, npm -v检测是否安装成功

本地博客搭建

  1. 在本地创建一个文件夹用于存放博客的相关文件(存放位置自定义)

​ 命令行输入：makir blog

  1. 进入blog文件，使用hexo初始化博客

    命令行输入：hexo init

​ 初始化完成后会看到提示: start blogging with hexo

  1. 使用本地预览命令启动博客

    命令行输入：hexo s

  2. 新建一篇博文

​ hexo -n "blog name"，之后可以使用任意markdown编辑器书写博文

  1. 生成博文

​ hexo g

随后便可在本地博客查阅新建的博文。

部署博客

推荐使用github部署博客

  1. 在github新建仓库，注意仓库名称必须为github用户名，即<username>.github.io。
  2. 在本地blog文件下安装插件npm install --save hexo-deployer-git
  3. 打开blog/_config.yml文件，增加Deployment部分
  1. 部署到远端

​ hexo d

  1. 访问博客，使用https://username.github.io

这样一个属于自己的博客就建成啦！

密钥配置

​ 如果之前没有在github上提交过项目，则需要配置密钥

  1. 设定git账户和邮箱，其中 username 是你的用户名，email@gmail.com是你github的登录邮箱
  1. 然后通过终端命令生成SSH Key
  1. 复制SSH Key
  1. 打开并登陆github，点击用户头像>settings>SSH and GPG keys>New SSH key 粘贴刚刚复制的密钥，就可以将本地项目推送至远端啦。

常用命令

Usage: hexo

  • Commands:

    clean Remove generated files and cache.

    config Get or set configurations.

    deploy Deploy your website.

    generate Generate static files.

    help Get help on a command.

    init Create a new Hexo folder.

    list List the information of the site

    migrate Migrate your site from other system to Hexo.

    new Create a new post.

    publish Moves a draft post from _drafts to _posts folder.

    render Render files with renderer plugins.

    server Start the server.

    version Display version information.

  • Global Options:

    –config Specify config file instead of using _config.yml

    –cwd Specify the CWD

    –debug Display all verbose messages in the terminal

    –draft Display draft posts

    –safe Disable all plugins and scripts

    –silent Hide output on console
